# Implementation Plan: Language Switcher (Troca de Idioma)

**Branch**: `001-language-switcher` | **Date**: 2026-07-03 | **Spec**: [spec.md](file:///C:/Users/Eduardo/Documents/GitHub/portfolio/specs/001-language-switcher/spec.md)

**Input**: Feature specification from `/specs/001-language-switcher/spec.md`

**Note**: This plan is filled in based on the `/speckit-plan` workflow.

## Summary

Implement a global language toggle mechanism (EN/PT) leveraging a React Context (`LanguageProvider`), LocalStorage preference persistence, static UI dictionaries, and parallel CMS content fetching via GraphQL query aliasing to ensure instant translation and zero loading flicker while preserving SEO.

## Technical Context

**Language/Version**: TypeScript ^5.1.0, Node.js 18+

**Primary Dependencies**: Next.js ^16.2.6 (App Router), React 18.2.0, Framer Motion 10.11.2, Tailwind CSS 3.3.1

**Storage**: localStorage (client preference persistence)

**Testing**: Jest ^30.4.0, React Testing Library

**Target Platform**: Node.js / Vercel

**Project Type**: Web application

**Performance Goals**: UI language toggle updates < 100ms, page load time under 1.5s

**Constraints**: Same URL routing (no language subroutes). Must query both Portuguese and English versions of dynamic data from Hygraph at build/revalidation time and render the selected version based on active locale.

**Scale/Scope**: 1 user toggle, 2 locales (PT-BR, EN), portfolio scope

## Constitution Check

*GATE: Must pass before Phase 0 research. Re-check after Phase 1 design.*

- **ISR/SSR Caching (Gate I):** We MUST ensure that parallel fetching of both languages on the server maintains Next.js App Router caching mechanism (`60 * 60 * 24` hours caching for Hygraph queries). (Status: PASSED)
- **headless CMS Integration (Gate II):** All localized data MUST be fetched from Hygraph using native locales, avoiding any frontend hardcoding of projects or experiences. (Status: PASSED)
- **Visual Excellence (Gate III):** The language selector button must match the site's sleek theme, utilizing subtle micro-animations (Framer Motion). (Status: PASSED)
- **Testing Discipline (Gate IV):** Automated Jest/Testing Library unit tests must be added to cover the Language Selector and Language Context logic. (Status: PASSED)
- **Clean Architecture & Types (Gate V):** Local dictionaries must be structured cleanly. Data models in `app/types` must explicitly type all aliased query outputs. (Status: PASSED)
